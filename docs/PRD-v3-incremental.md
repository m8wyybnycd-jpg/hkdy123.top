## 二、产品目标

### 增量目标

1. **邮箱验证码注册**：将注册流程从"用户名+密码"升级为"邮箱+验证码+密码"，确保用户邮箱真实性，为后续密码找回、消息通知等功能奠定基础
2. **后台管理系统**：为站长提供可视化的管理后台，支持数据查看、用户管理、内容管理、权限管理，提升运营效率
3. **安全增强**：验证码频率限制 + 有效期控制，防止恶意注册和短信轰炸

### 与 V2.0 PRD 目标的关系

| V2.0 目标 | V3.0 变化 |
|-----------|-----------|
| 门控访问（账号系统） | **增强** — 从用户名注册升级为邮箱验证码注册 |
| 全平台覆盖 | 不变 |
| 薅羊毛聚合 | 不变 |
| —（新增） | **新增** — 后台管理系统，对应 V2.0 PRD 中 P2-3 和 P2-5 的升级实现 |

| ID | 需求 | 标注 | 描述 | 验收标准 |
|----|------|------|------|----------|
| P0-1 | 邮箱验证码注册 | 🔄 | 注册页从"用户名+密码"改为"邮箱+验证码+密码+确认密码"。用户输入邮箱 → 点击发送验证码 → 后端生成 6 位数字验证码 → 通过邮件服务发送 → 用户输入验证码 + 密码 + 确认密码 → 完成注册 | ① `POST /api/register` 参数改为 `{ email, code, password }` ② 验证码校验通过后才能注册 ③ 验证码 5 分钟过期 ④ 同一邮箱 60 秒内不可重复发送 ⑤ 注册成功后自动登录（返回 JWT） |
| P0-2 | 验证码发送接口 | 🆕 | 新增 `POST /api/send-code` 接口，接收邮箱地址 → 生成 6 位数字验证码 → 存入 D1 → 调用邮件服务发送 → 返回发送结果 | ① 接口接收 `{ email }` ② 60 秒频率限制（同一邮箱） ③ 验证码存入 `verification_codes` 表 ④ 邮件发送失败返回明确错误信息 ⑤ 注册用户已存在的邮箱也允许发送（用于后续密码找回扩展） |
| P0-3 | 邮箱密码登录 | 🔄 | 登录方式从"用户名+密码"改为"邮箱+密码"（与注册方式统一） | ① `POST /api/login` 参数改为 `{ email, password }` ② 根据 email 查找用户并校验密码 ③ 其余登录逻辑不变 |
| P0-4 | 验证码存储表 | 🆕 | D1 新增 `verification_codes` 表，存储验证码及过期时间 | ① 字段：email, code, expires_at, created_at, used ② 5 分钟过期自动清理（前端校验 + 后端校验双重保障） |
| P0-5 | 邮件服务集成 | 🆕 | 通过 Cloudflare Workers 原生调用 Mailchannels API 发送验证码邮件（`https://api.mailchannels.net/tx/v1/send`）。零配置、零 API Key、完全免费。Workers 运行在 CF 全球边缘网络，不受国内网络限制。 | ① 验证码邮件模板包含：6 位验证码、有效期提示（5 分钟） ② 邮件发送成功率 ≥95% ③ 无需任何 API Key 配置 |
| P0-6 | users 表扩展 | 🔄 | `users` 表新增 `email` 字段（替代原 username 作为唯一标识）和 `is_admin` 字段（管理员标识），保留 `username` 作为昵称字段 | ① email 字段 UNIQUE NOT NULL ② is_admin 字段 INTEGER DEFAULT 0 ③ username 字段改为可选（可后续在个人设置中填写） |
| P0-7 | 后台管理路由 + 布局 | 🆕 | 新增 `/admin` 路由，仅 `is_admin=1` 用户可访问。经典后台布局：左侧深色导航栏 + 顶部栏 + 右侧主内容区 | ① `/admin` 路由守卫检查 is_admin ② 非管理员访问重定向到首页 ③ 左侧导航可折叠 ④ 移动端左侧导航变为抽屉式 |
| P0-8 | 后台用户管理 | 🆕 | 管理员可查看所有注册用户列表，支持搜索（邮箱）、分页、查看详情（邮箱、注册时间、状态） | ① 用户列表页 ② 搜索框按邮箱模糊搜索 ③ 分页（每页 20 条） ④ 显示字段：邮箱、注册时间、管理员标识 |
| P0-9 | 后台登录 + 管理员识别 | 🔄 | 管理员使用前台邮箱+密码登录后，访问 `/admin` 时后端通过 JWT 中携带的 `is_admin` 字段判断权限。若无管理员权限则拦截 | ① JWT payload 新增 `is_admin` 字段 ② 登录接口返回时附带 `is_admin` ③ `/admin` 路由和后台 API 双重校验 is_admin |

## 七、增量数据表设计

### 7.1 verification_codes 表（新增）

```sql
-- 验证码表
CREATE TABLE IF NOT EXISTS verification_codes (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  email      TEXT    NOT NULL,
  code       TEXT    NOT NULL,          -- 6 位数字验证码
  expires_at TEXT    NOT NULL,          -- 过期时间（ISO 8601，创建时间 + 5 分钟）
  created_at TEXT    NOT NULL DEFAULT (datetime('now')),
  used       INTEGER NOT NULL DEFAULT 0 -- 0=未使用, 1=已使用
);

-- 索引：按邮箱查询验证码（用于校验）
CREATE INDEX IF NOT EXISTS idx_vcodes_email ON verification_codes(email);

-- 索引：清理过期验证码
CREATE INDEX IF NOT EXISTS idx_vcodes_expires ON verification_codes(expires_at);
```

**字段说明**：
- `email`：接收验证码的邮箱地址
- `code`：6 位数字验证码（000000-999999）
- `expires_at`：有效期 5 分钟，前端和后端双重校验
- `used`：标记是否已被使用，防止同一验证码重复使用

### 7.2 users 表变更（修改）

**V2.0 原表结构**：
```sql
CREATE TABLE IF NOT EXISTS users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  username      TEXT    UNIQUE NOT NULL,
  password_hash TEXT    NOT NULL,
  salt          TEXT    NOT NULL,
  created_at    TEXT    NOT NULL DEFAULT (datetime('now'))
);
```

**V3.0 迁移 DDL**：
```sql
-- Step 1: 新增 email 字段（先允许 NULL，填充后再改为 NOT NULL）
ALTER TABLE users ADD COLUMN email TEXT;
ALTER TABLE users ADD COLUMN is_admin INTEGER NOT NULL DEFAULT 0;

-- Step 2: 为存量用户生成占位邮箱（username@legacy.local）
UPDATE users SET email = username || '@legacy.local' WHERE email IS NULL;

-- Step 3: 添加 email 唯一约束
-- 注意：D1 SQLite 不支持直接 ADD CONSTRAINT，需要重建表或使用 CREATE UNIQUE INDEX
CREATE UNIQUE INDEX IF NOT EXISTS idx_users_email ON users(email);

-- Step 4: 设置默认管理员（根据环境变量或手动指定）
-- UPDATE users SET is_admin = 1 WHERE email = 'admin@example.com';
```

**V3.0 目标表结构**：
```sql
CREATE TABLE IF NOT EXISTS users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  email         TEXT    UNIQUE NOT NULL,  -- 🆕 邮箱（替代 username 作为登录标识）
  username      TEXT,                     -- 🔄 改为可选（昵称）
  password_hash TEXT    NOT NULL,
  salt          TEXT    NOT NULL,
  is_admin      INTEGER NOT NULL DEFAULT 0, -- 🆕 管理员标识
  created_at    TEXT    NOT NULL DEFAULT (datetime('now'))
```
